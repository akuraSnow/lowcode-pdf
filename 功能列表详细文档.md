## 1. 编辑器核心功能

### 1.1 编辑模式管理

#### 模式类型

- **编辑模式 (EDIT)**
  - 允许添加、删除、编辑组件
  - 显示选中边框和控制点
  - 显示辅助线和对齐线
  - 显示工具栏和操作按钮
- **预览模式 (PREVIEW)**
  - 隐藏所有编辑工具
  - 隐藏选中边框
  - 显示最终渲染效果
  - 支持交互功能测试
- **发布模式 (PUBLISH)**
  - 生成生产环境代码
  - 导出静态资源
  - 生成独立访问链接

#### 模式切换

- 顶部工具栏快速切换
- 快捷键支持（待实现）
- 状态保持和恢复

---

### 1.2 布局模式管理

#### 瀑布流布局 (Waterfall)

- **特性**
  - 组件自上而下流式排列
  - 自动计算组件位置
  - 支持组件拖拽排序
  - 响应式宽度适配

- **配置项**
  - `order`: 组件排序索引
  - `width`: 宽度（px / % / 'fill'）
  - `margin`: 组件间距
  - `align`: 对齐方式（left/center/right）
  - `display`: 显示方式（block/inline-block/inline）

#### 绝对定位布局 (Absolute)

- **特性**
  - 组件自由定位
  - 精确像素控制
  - 支持重叠布局
  - 支持百分比定位

- **配置项**
  - `position.left/right/top/bottom`: 定位坐标（px / %）
  - `size.width/height`: 组件尺寸
  - `rotation`: 旋转角度
  - `scale`: 缩放比例

#### 布局模式切换

- 一键切换布局模式
- 自动转换组件配置
- 保持组件相对位置关系

### 3.2 组件类型详解

#### 文本组件 (Text)

- **基础属性**
  - content: 文本内容
  - fontSize: 字体大小 (12-72px)
  - color: 文字颜色
  - fontWeight: 字体粗细 (normal/bold/100-900)
  - textAlign: 对齐方式 (left/center/right/justify)

- **文本样式**
  - bold: 加粗
  - italic: 斜体
  - underline: 下划线
  - strikethrough: 删除线

- **高级样式**
  - fontFamily: 字体家族
  - lineHeight: 行高
  - letterSpacing: 字间距
  - textIndent: 首行缩进
  - textTransform: 文本转换 (uppercase/lowercase/capitalize)
  - whiteSpace: 空白处理 (normal/pre/nowrap/pre-wrap)

- **装饰样式**
  - backgroundColor: 背景颜色
  - borderColor: 边框颜色
  - borderWidth: 边框宽度
  - borderStyle: 边框样式 (solid/dashed/dotted)
  - padding: 内边距

- **特殊功能**
  - 富文本编辑
  - 文本范围逻辑（对特定文本片段设置规则）
  - 数据绑定（动态显示数据）
  - 内联编辑

#### 表格组件 (Table)

- **列配置**
  - key: 列唯一标识
  - title: 列标题
  - dataIndex: 数据字段名
  - width: 列宽度
  - align: 对齐方式 (left/center/right)
  - render: 自定义渲染函数

- **多行表头**
  - children: 子列配置
  - colSpan: 跨列数
  - 无限层级嵌套
  - 自动计算表头高度

- **表格样式**
  - headerStyle: 表头样式配置
  - cellStyle: 单元格样式配置
  - bordered: 显示边框
  - tableSize: 表格大小 (small/middle/large)

- **单元格管理**
  - 单元格样式定制
  - 单元格内容自定义
  - 单元格合并（rowSpan/colSpan）
  - 单元格逻辑规则
  - 单元格数据绑定

- **数据管理**
  - dataSource: 静态数据
  - 数据绑定: 动态数据
  - pagination: 分页配置
  - 数据过滤（待实现）
  - 数据排序（待实现）

- **特殊功能**
  - 单元格编辑器
  - 单元格选择器
  - 智能分页（PDF导出）
  - 表格模板

#### 图片组件 (Image)

- **基础属性**
  - src: 图片地址
  - alt: 替代文本
  - fit: 适应方式
    - contain: 包含（保持比例）
    - cover: 覆盖（裁剪）
    - fill: 填充（拉伸）
    - none: 原始大小
    - scale-down: 缩小

- **功能**
  - 图片上传
  - 资源库选择
  - 外链地址输入
  - 图片预览
  - 图片裁剪（待实现）

#### 图表组件 (Chart)

- **图表类型**
  - bar: 柱状图
  - line: 折线图
  - pie: 饼图
  - scatter: 散点图

- **配置项**
  - chartType: 图表类型
  - data: 图表数据
  - option: ECharts配置对象
  - 响应式尺寸

- **数据源**
  - 静态数据配置
  - API数据绑定

  - 实时数据更新

#### 条形码组件 (Barcode)

- **属性**
  - value: 条形码值
  - format: 编码格式

    - EAN13: 商品条码
    - UPC: 美国统一代码
  - lineWidth: 线宽
  - height: 高度
  - displayValue: 显示文字

#### 二维码组件 (QRCode)

#### 容器组件 (Frame)

### 3.3 容器布局系统

#### 水平布局 (Horizontal)

- **特性**
  - 子组件横向排列
  - 自动计算宽度分配
  - 支持宽度类型设置

- **配置项**
  - layout: 'horizontal'
  - gap: 子组件间距 (0-100px)
  - 子组件配置:
    - widthType: 'px' | 'flex'
    - widthValue: 具体值

- **宽度分配**
  - px: 固定像素宽度
  - flex: 弹性比例分配
  - 混合模式: 固定+弹性

#### 垂直布局 (Vertical)

- **特性**
  - 子组件纵向排列
  - 自动计算高度分配
  - 支持高度类型设置

- **配置项**
  - layout: 'vertical'
  - gap: 子组件间距 (0-100px)
  - 子组件配置:
    - heightType: 'px' | 'flex'
    - heightValue: 具体值

- **高度分配**
  - px: 固定像素高度
  - flex: 弹性比例分配
  - auto: 内容自适应

#### 固定定位 (Fix)

- **特性**
  - 子组件绝对定位
  - 自由放置位置
  - 支持重叠布局

- **配置项**
  - layout: 'fix'
  - 子组件配置:
    - position.left/right/top/bottom
    - 支持 px 和 % 单位

- **应用场景**
  - 复杂布局
  - 精确定位
  - 层叠效果

#### 网格布局 (Grid)

- **特性**
  - 网格自动布局
  - 子组件平均分配
  - 响应式适配

- **配置项**
  - layout: 'grid'
  - gridColumns: 列数 (1-12)
  - gridRows: 行数 (1-20)
  - gap: 网格间距 (0-50px)

- **行为**
  - 子组件按顺序填充网格
  - 自动换行
  - 等宽等高单元格

#### 表格布局 (Table)

- **特性**
  - 类似表格的布局
  - 精确控制行列
  - 支持跨行跨列（待实现）

- **配置项**
  - layout: 'table'
  - tableColumns: 列数
  - tableRows: 行数
  - gap: 单元格间距

- **与Grid区别**
  - 更严格的行列控制
  - 支持空单元格
  - 表格语义化

### D. 开发指南

#### 添加新组件类型

1. 在 `types/index.ts` 添加组件接口
2. 在 `ComponentPanel` 添加组件定义
3. 在 `ComponentRenderer` 添加渲染逻辑
4. 在 `PropertyPanel` 添加属性编辑UI
5. 更新文档和示例

#### 添加新逻辑动作

1. 在 `types/index.ts` 的 `ActionType` 枚举添加
2. 在 `hooks/useRuleEngine.ts` 实现动作逻辑
3. 在 `RuleEditor` 添加UI配置
4. 测试和文档更新

#### 性能优化建议

- 使用 `React.memo` 避免不必要的重渲染
- 使用 `useMemo` 和 `useCallback` 缓存计算结果
- 大列表使用虚拟滚动
- 图片懒加载
- 组件按需加载
