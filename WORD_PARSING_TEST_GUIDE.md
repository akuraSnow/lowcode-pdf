## 📋 测试场景清单

### 测试场景 1：基础 Word 文档导入

**测试步骤**：
1. 在编辑器中找到"资源管理器"或"插件"菜单中的"Word 文档导入"功能
2. 选择上传一个 `.docx` 或 `.doc` 文件（必须包含至少一段文本）
3. 预览对话框应该打开并显示文档内容
4. 点击"应用"按钮

**期望结果** ✅：
- [ ] 文档正确解析，预览显示内容
- [ ] 预览对话框显示清晰的元素结构
- [ ] 点击应用后，内容成功添加到画布
- [ ] 显示成功信息，如"已添加 X 个元素"

**如果失败**：
- 检查浏览器控制台是否有错误信息
- 查看添加后画布是否有任何内容
- 记录具体的错误消息

---

### 测试场景 2：编辑预览内容

**测试步骤**：
1. 上传一个包含文本的 Word 文档
2. 在预览对话框中，修改其中一段文本（例如，更改颜色或内容）
3. 点击"应用"按钮

**期望结果** ✅：
- [ ] 预览中修改的文本显示为修改后的内容
- [ ] 点击应用后，修改的内容添加到画布
- [ ] 修改不会丢失（例如，如果修改了颜色，画布上应该显示修改后的颜色）

**关键验证点**（这是修复的核心）：
- [ ] **样式保留**：如果原文本有颜色/字体大小，修改后仍然保留
- [ ] **属性完整**：编辑后的元素所有属性都被保留，不丢失任何信息

## 🆘 常见问题排查

### Q1：上传后没有反应
**可能原因**：
- 文件格式不支持
- 文件损坏
- 网络问题

**解决方案**：
- 检查浏览器控制台错误
- 尝试使用其他 Word 文件
- 检查网络连接

### Q2：预览显示但应用失败
**可能原因**：
- Canvas 获取失败
- importSchema 执行错误

**解决方案**：
- 查看控制台中的"导入Schema失败"错误
- 检查画布是否正确初始化
- 查看 Canvas 的 root 节点是否存在

### Q3：应用后某些样式丢失
**可能原因**：
- 浅拷贝未能保留嵌套属性（这是修复的主要问题）

**解决方案**：
- 确保已应用深拷贝修复
- 重新构建应用
- 清除浏览器缓存

### Q4：元素计数不正确
**可能原因**：
- flattenSchema 函数未正确展平
- 某些元素无效或为空

**解决方案**：
- 检查控制台中的导入日志
- 验证 flattenSchema 的输出
- 检查 Schema 结构是否正确
