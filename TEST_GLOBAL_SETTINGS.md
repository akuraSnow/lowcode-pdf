### 测试场景 1: 配置全局路径

**目标**: 验证全局设置对话框功能

1. 打开主页
   ```
   http://localhost:5556/home.html
   ```

2. 检查页面元素
   - ✅ 应该看到"全局设置"按钮（右上角）
   - ✅ 产品表格中**没有**"保存路径"列
   - ✅ 有"新建产品"按钮

3. 点击"全局设置"按钮
   - ✅ 弹出对话框标题: "全局路径设置"
   - ✅ 看到4个输入框:
     * 产品JSON保存路径
     * 模板保存路径
     * 方法/组件保存路径
     * 通用文件保存路径
   - ✅ 默认值应该是:
     * `./data/products`
     * `./data/templates`
     * `./data/methods`
     * `./data/files`

4. 修改路径
   ```
   产品JSON保存路径: ./test/products
   模板保存路径: ./test/templates
   方法/组件保存路径: ./test/methods
   通用文件保存路径: ./test/files
   ```

5. 点击"保存设置"
   - ✅ 对话框关闭
   - ✅ 显示成功消息: "全局设置已保存"

6. **验证持久化**: 刷新页面
   - 重新打开"全局设置"
   - ✅ 路径应该保持为 `./test/*`（localStorage持久化生效）

7. 测试"恢复默认"按钮
   - 点击"恢复默认"
   - ✅ 所有路径恢复为 `./data/*`
   - 点击"保存设置"
   - ✅ 默认值被保存

### 测试场景 2: 创建产品（无需保存路径）

**目标**: 验证产品创建表单已移除savePath字段

1. 点击"新建产品"按钮
   
2. 检查表单字段
   - ✅ 有"产品名称"输入框
   - ✅ 有"产品描述"文本域
   - ✅ **没有**"JSON保存路径"字段
   
3. 填写产品信息
   ```
   产品名称: 测试全局路径产品
   产品描述: 测试使用全局路径配置的产品
   ```

4. 点击"创建并编辑设计"
   - ✅ 进入编辑器页面
   - ✅ 产品数据不包含 `savePath` 字段

### 测试场景 3: 发布功能（使用全局路径）

**目标**: 验证发布时自动使用全局路径 + 产品名称

1. **配置全局路径** (如果未配置)
   - 回到主页: http://localhost:5556/home.html
   - 点击"全局设置"
   - 设置产品路径: `./test-output/products`
   - 保存设置

2. **创建测试目录** (首次测试需要)
   ```bash
   mkdir -p ./test-output/products
   ```

3. **创建或打开产品**
   - 创建新产品: `全局路径测试`
   - 或编辑现有产品

4. **在编辑器中编辑设计**
   - 添加一些组件
   - 修改页面内容

5. **点击"发布"按钮** (右上角)
   - ✅ 弹出确认对话框
   - ✅ 对话框显示路径: 
     ```
     确认发布到: ./test-output/products/全局路径测试.json
     ```
   - 注意: 路径是 `全局路径 + 产品名称.json`

6. **确认发布**
   - 点击"确定"
   - ✅ 显示成功消息: "发布成功！"

7. **验证文件创建**
   ```bash
   ls -la ./test-output/products/
   cat ./test-output/products/全局路径测试.json
   ```
   - ✅ 应该看到 `全局路径测试.json` 文件
   - ✅ 文件包含完整的页面schema

### 测试场景 4: 错误处理

**目标**: 验证未配置路径时的错误提示

1. **清除localStorage** (模拟首次使用)
   - 打开浏览器DevTools (F12)
   - Console中执行:
     ```javascript
     localStorage.removeItem('global-settings-storage');
     location.reload();
     ```

2. **创建产品并尝试发布**
   - 创建新产品: `错误测试产品`
   - 进入编辑器
   - 点击"发布"

3. **验证错误提示**
   - ✅ 应该显示错误消息: 
     ```
     请先在主页配置全局路径设置！
     ```
   - ✅ 发布操作被阻止

4. **修复: 配置全局路径**
   - 返回主页
   - 点击"全局设置"
   - 配置路径并保存
   - 回到编辑器
   - 再次点击"发布"
   - ✅ 发布成功

### 测试场景 5: 路径验证

**目标**: 验证路径格式验证

1. 打开"全局设置"对话框

2. **测试无效路径**
   - 输入: `data/products` (缺少 `./`)
   - ✅ 应该显示错误提示: "路径必须以 ./ 开头"
   - ✅ "保存设置"按钮禁用

3. **测试特殊字符**
   - 输入: `./data/产品#路径`
   - ✅ 应该显示错误提示: "路径只能包含字母、数字、下划线、横线和斜杠"

4. **测试有效路径**
   - 输入: `./my-data/products_v2`
   - ✅ 无错误提示
   - ✅ "保存设置"按钮启用
